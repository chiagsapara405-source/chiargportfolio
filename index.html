What I Learned

Technical Insights

Problems I ran into, choices I made, and what I took away from them.

💡 Tech Choice
Why I used a struct for the Library System
In C, I needed to store multiple details about each book together — the ID, name, author, quantity and price. I learned about structs which let you group different types of data into one unit. It made the code much cleaner than using separate arrays for each field.
What I took from it
Structs taught me to think about how to organise data before writing functions. That idea — design your data first — is something I keep using.
🐛 Bug I Fixed
Book names with spaces weren't saving correctly
When I used scanf("%s") to read book names, it would stop at the first space. So "Program With C" got saved as just "Program". Took me a while to figure out why the data was wrong.
How I fixed it
My teacher told me to use scanf(" %[^\n]", name) instead — this reads the whole line including spaces. The space before % also clears leftover input from before. Simple fix but I learned a lot about how scanf actually works.
💡 Tech Choice
Why I used Lovable for the Restaurant App
I wanted to build something with a proper UI — QR codes, an admin panel — but I didn't have the JavaScript skills yet to build it from scratch. Lovable let me describe what I wanted and helped generate it.
What I took from it
I still had to make decisions about the flow and features. It showed me what a real web app looks like under the hood and motivated me to keep learning JS properly so I can build things like this myself eventually.
📈 Growth
Moving from C to HTML and CSS
Coming from C, HTML felt easy at first — just tags. But making things look good and work on mobile was harder than expected. Flexbox confused me for a while. I kept having to look things up.
Where I'm at now
I built this portfolio page to practice. I looked up how to do scroll animations, how to make a navbar sticky, and how to make it work on mobile. Still learning, but building real things is the best way I've found to actually improve.
